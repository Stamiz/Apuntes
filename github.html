Ramas

Main

Aqui no se programa. Solo se hace mantenimiento. Lo que se haya subido a main no se toca y se sube cada varias iteraciones cuando se tiene algo cerrado de verdad. Algún avance importante.

Development

Aqui es donde se programa de verdad. Con los conocimientos adquiridos en "Experimental". Lo que se ha probado ahí se reprograma aquí. Lo que se desarrolle en Development no puede durar mas de 2 dias de desarrollo. Tiene que estar bien atomizado. Está prohibido copiar de Experimental a Development

Experimental

Probar probar y más probar. Se hace una rama en local que es una copia y ya a aprender reventando el código. En cada iteración se va a eliminar todo lo que exista aquí y se va a llenar Experimental con lo que exista en Development. Aquí se hacen funciones rápidas, para probar cosas rápido. Es la parte práctica del aprendizaje. Imporante tomar apuntes para aprender y aplicarlo en Development. Aplicar el conocimiento, no el código.

Tareas

Nunca se hacen en una rama. Las tareas siempre en una copia de Experimental. Se mete a Experimental una vez terminada la tarea y se sube a Experimental
